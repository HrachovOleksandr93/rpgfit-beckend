# Portals Vision

Тема порталів — нова, у бекенд-коді її ще нема. Документ фіксує концепт, щоб
BA-агенти планували її як **окремий epic**, а не дострибували до реалізації.

## Контекст з лору

- У «Теорії Світу» тріщини — це наслідок Дня Розколу (31.10.2042). ~4700
  порталів по світу, зростає на ~40/місяць, усього зараз ~6200.
- Кожна тріщина прив'язана до 1+ реалмів (Олімп / Асгард / ...).
- «Вектор» (застосунок у грі = наш телефон) може сканувати тріщину й
  визначити реалм, клас сутностей і доступні завдання.

## Два типи порталів (бачення)

### A. Dynamic portals — динамічні тріщини

- **Що це:** тимчасові локації на карті, які з'являються через алгоритм
  (біля user-локації, біля значущих точок, у відповідь на активність юзерів).
- **Geolocation:** на мапі, аналог «портали Ingress» або «спавни Pokémon GO».
- **Поведінка:** можна навідатись, запустити Battle → моби з'являються за
  реалмом порталу.
- **Створення юзером:** юзер може створити власний портал у «особистих
  локаціях» — за допомогою спеціального айтема (див. `onboarding-gifts.md`).
  Створений портал має час життя / обмеження на кількість пройдених Battle.

### B. Static portals — статичні портали у відомих місцях

- **Що це:** постійні локації на реальних landmarks (гори, храми, стадіони,
  каньйони), вручну додані адміном.
- **Приклади з лору §8.3 (артефакти):** вершина Галдхьопіген, дно Санторіні,
  лабіринт печер Дікті, ущелина Пінду, Ватнайокутль, Ерг-Шебі, Дніпро біля
  Канева, Амазонія, Гімалаї, Теотіуакан, Ангкор-Ват, Крим-Яйла.
- **Ідея фаундера:** прив'язати до таких локацій **тренувальні виклики**.
  Приходить юзер → портал видає «challenge-місію»: пробігти X km,
  відспортсменити Y часу, або виконати серію вправ.
- **Нагорода:** артефакт відповідного Tier (1–3), есенція, XP-бонус,
  унікальна косметика («я тут був»).
- **Доступність:** не всі юзери поряд з Гімалаями. Тому треба теж «virtual»
  версії тих самих порталів — полегшена репліка (нижча нагорода, той самий
  сюжет).

## Зв'язки з іншими доменами

| Домен | Залежність |
|-------|------------|
| Mob system | моб спавниться з `portal.realm` |
| Artifact/Item | портал → артефакт як дроп |
| Battle | `Battle.source = portal_id` як новий тип запуску |
| Health | workout challenge = перевіряємо виконання по HealthKit даних |
| Geolocation | потрібен новий модуль: lat/lng, радіус, геофенс, push-нотифікації |

## MVP-варіант (для beta)

Не пропонуємо одразу всю систему. Для beta достатньо:

1. **10–20 статичних порталів** біля великих міст України, ЄС і США.
2. Для кожного — 1 простий workout-challenge, 1 rare-артефакт як нагорода.
3. Віртуальна версія кожного з порталів — тренувальний еквівалент (без геолокації).
4. Dynamic portals і user-created portals — **відкладаємо на 1.0+**.

## Відкриті питання

- Хто модерує статичні портали? (очевидно — адмін Sonata).
- Чи треба портали видимі, якщо юзер далеко? Чи показувати тільки у радіусі X км?
- Як балансувати нагороду: глобальна (тільки перші 1000 здобудуть
  оригінальний артефакт) чи індивідуальна (кожен може отримати окремо)?
- Права на landmarks: чи є legal-ризик згадувати Дніпро, Ватнайокутль у
  застосунку? (Зазвичай — ні, геоназви публічні; але згадка конкретного храму
  як «ти тут знайдеш Трішулу» — треба перевірити).
- Чи треба AR-камера в статичних порталах? (Pokémon GO-style)
