# RPGFit — Vision & Product Ideas

Ця папка — «місце, де живуть ідеї фаундера», до того як вони перетворюються
на специфікації в `rpgfit-beckend/docs/BUSINESS_LOGIC.md` або на плани в
`docs/superpowers/plans/`.

Правило з пам'яті: **всі бізнес-правила документуємо в docs/, а не тільки в коді**.
Цю папку читають BA-агенти з `BA/` і розробка.

## Зміст

| Файл | Про що |
|------|--------|
| `mobs.md` | Бачення монстрів / ворогів, зміни до існуючої mob-системи |
| `portals.md` | Теорія порталів: динамічні тріщини + статичні портали в відомих місцях |
| `onboarding-gifts.md` | Подарунки за перший вхід, у т.ч. предмет створення порталу |
| `beta-hype.md` | Як вести beta, щоб була хвиля (заготовка — агент 02 дозаповнить) |
| `emotional-hooks.md` | Які емоції культивуємо + червоні лінії (grind, pay-to-win) + канали монетизації |

## Статус

Все в цій папці — **чорновик ідеї**. Не вважати як затверджену фічу, поки не
описано в `BUSINESS_LOGIC.md` бекенду або у спеці. BA-агенти беруть звідси
вхід, але мають явно позначати `[гіпотеза]` у звітах.
