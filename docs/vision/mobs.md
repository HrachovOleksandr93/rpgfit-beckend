# Monsters Vision

## Поточний стан (як сьогодні в коді)

Див. `rpgfit-beckend/docs/BUSINESS_LOGIC.md §10 Mob System`:

- ~2000 мобів (20 на рівень × 100 рівнів).
- HP `= 20 · level^1.5 + 40 ± 20%`.
- XP `= (4.2·level² + 28·level) / 15 · rarity_multiplier ± 10%`.
- Рідкість: common / uncommon / rare / epic / legendary.
- Відбір моба у бою: `level ± 2`, фільтр за режимом (Custom/Recommended/Raid).

## Бачення (зміни від фаундера)

> Будуть зміни по монстрам з якими сражаться персонажі.

Зміни не фіналізовані, але напрямок — **прив'язка мобів до реалмів з лору**
(Олімп, Асгард, Дхарма, Ду'ат, Нав, Шиба) замість generic-істот.

### Пропоновані осі для пере-дизайну

1. **Realm affiliation** — додати поле `Mob.realm` (enum: olympus, asgard,
   dharma, duat, nav, shiba, neutral). Впливає на:
   - візуал моба (скін, ефекти)
   - вразливість до артефактів: артефакт реалму X робить +40% урону по
     мобах реалму X (узгоджено з лором §8.1 «Прив'язка до реалму»)
   - сюжетну прив'язку — моби з'являються біля тріщин своїх реалмів

2. **Class-based замість level-based нарахування** — лор виділяє 4 класи:
   I Слабкі (нечисть), II Середні (напівбоги), III Сильні (малі боги),
   IV Елітні (Олімпійці, Аси, Деви першого рівня).
   - Mapping: common+uncommon → Class I; rare → Class II; epic → Class III;
     legendary → Class IV.
   - Класи визначають кількість «Завдань Оракула» (можна трактувати як
     cадіє / окремий бойовий тик).

3. **Mob behavior type** — не тільки «б'єш палкою», а й:
   - `physical` — треба сирий урон (поточна механіка),
   - `oracle_task` — треба виконати фізичне завдання («зроби 20 підтягувань
     за цей сеанс» — лор §9.3),
   - `ritual` — знання лору або спец-ітем,
   - `team` — Class III+ потребують рейду.

4. **Motivation / flavor text** — кожен моб має 1–2 рядки лор-тексту
   («Йотун з Асгарду, вигнаний після Рагнароку»). Сьогодні в `Mob`
   є `description`, але порожнє.

### Що не треба чіпати

- HP/XP формули — вони сходяться з leveling curve (~15 мобів = рівень).
  Не чіпати без перерахунку всього leveling-документу.
- CSV-імпорт `app:import-mobs` і схема `data/mobs.csv` — розширити новими
  колонками, а не ламати старі.

## Відкриті питання

- Чи всі 2000 мобів переробляємо на realm-based, чи лишаємо «neutral»
  групу для рівнів 1–5 як easy-start?
- Bosses (Class IV = legendary) — одноразові чи респавняться?
- Чи виходять Class IV тільки в статичних порталах (див. `portals.md`),
  чи в звичайних тріщинах теж?
