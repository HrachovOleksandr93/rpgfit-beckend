# Onboarding Gifts

Ідея фаундера: **при першому вході гравець отримує подарунок** — це підвищує
D1 retention і формує перший емоційний hook.

## Стартовий пакет (вхідний)

### 1. Portal Creation Kit (ключовий айтем)

- Дозволяє створити 1 персональний портал у власній локації (дім / спортзал /
  парк).
- Портал має обмеження: N Battle-сеансів або M днів життя.
- Рідкісний варіант з events: кількаразовий kit, post-launch або за платні
  сезони.
- **Чому важливо:** дає юзерові «свій куточок» гри з першого дня, не чекаючи
  поки дізнається, що таке глобальні тріщини. Базовий motivator — «це моє».

### 2. Starter Vector skin / cosmetic

- Косметика для UI персонажа (без статів). Індикація «я ранній юзер» — якщо
  отримано в перші N місяців, скін виглядає інакше.
- Аналог: launch-exclusive у Destiny 2, Pokémon GO launch avatar.

### 3. XP boost potion (24h)

- +50% XP від health-sync на перші 24 години. Щоб перша сесія була «сок».
- Не стекається з денним cap — cap теж підвищуємо на 24 години, або
  тимчасово знімаємо.

### 4. Starter artifact Tier 1

- Один Tier 1 artifact випадковий, одразу в інвентарі.
- Відкриває skill tree (зброя / магія / дистанційна) — вибір задає трохи
  стиль гри.

## Механіки видачі

- **Реєстрація → видача** — після `onboardingCompleted = true` сервіс
  `OnboardingService` видає стартовий пакет (розширюємо наявний service
  замість створення нового).
- **Тригери** — перший Battle (bonus loot), 7-day login streak (новий
  Portal Creation Kit), level 10 (skin upgrade).

## Beta-варіант

Мінімум для beta:
1. Portal Creation Kit × 1.
2. XP boost 24h.
3. Starter artifact (Tier 1, випадковий з 3).

Skin, streak-нагороди → 1.0.

## Відкриті питання

- Чи Portal Creation Kit можна дарувати другові (referral механіка)?
- Що робити, якщо юзер створює портал, але потім ніколи туди не
  повертається (мертвий портал — видаляємо автоматично через X днів).
- Видача артефакту випадкова чи юзер вибирає на онбордингу? (Другий варіант
  сильніше впливає на stat-build).
