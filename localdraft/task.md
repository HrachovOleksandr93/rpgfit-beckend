# ⚙️ Задачи для бэкенд-команды (Symfony) — День 1

## Цель на сегодня
Поднять инфраструктуру в Docker, спроектировать базовый слой домена для сущности пользователя и реализовать эндпоинт регистрации с покрытием тестами.

## Задачи

### 1. Инфраструктура и Docker
* Настроить `docker-compose.yml` в директории `web/`.
* Поднять связку: **PHP (Symfony) + Redis + MySQL**.
* Настроить тестовое окружение: использовать **SQLite** для быстрого прогона тестов.

### 2. Проектирование слоя данных (DDD)
* Создать и описать Entities (Сущности), Models, и DTO для процесса регистрации.
* Настроить **API Platform** для генерации контрактов и документации (Swagger/OpenAPI).
* **Важное правило для кода:** Комментарии в коде строго на английском. При модификации логики в будущем — не удалять существующий рабочий код без крайней необходимости, добавлять новые правила (например, для предотвращения infinity loop), расширяя логику, а не ломая старую.

### 3. Регистрация (Endpoint)
* Реализовать логику приема DTO с мобильного приложения.
* Валидация данных (проверка на уникальность логина/имени).
* Хэширование пароля и сохранение полной анкеты (рост, вес, раса, цели и т.д.) в БД.

### 4. Тестирование (PHPUnit)
* Проверить флоу регистрации POST-запросом (имитация запроса от мобильного приложения).
* Написать **Unit-тесты** для проверки маппинга DTO в Entity.
* Написать **Integration/Functional тесты** для проверки эндпоинта регистрации (запись в тестовую SQLite базу, проверка ответов 201 Created / 400 Bad Request).

**Ожидаемый результат:** Работающий в Docker бэкенд, принимающий JSON с анкетой пользователя, сохраняющий его в MySQL и проходящий зеленый билд в PHPUnit.
